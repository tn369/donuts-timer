# ペルソナ分析

## 1. 目的と利用範囲

本ドキュメントは、「どーなつタイマー」の機能検討・UI文言設計・受け入れ基準の判断軸をそろえるためのペルソナ分析です。  
親子での利用を前提に、順番管理・見通し形成・時間管理が難しい4歳児の日常タスク支援を中心に扱います。

### 非目的

- 医療的な診断や評価の代替を行うこと
- 特定の診断名だけで利用者像を固定化すること

## 2. 利用コンテキストの前提

- 主利用シーンは家庭内（朝の支度、帰宅後の宿題、就寝前ルーティン）
- 子ども単独操作に加え、保護者が設定や伴走を行う
- 視覚的な進捗表示と短い文言による支援を重視する
- データは LocalStorage のみで管理し、外部送信しない

## 3. ペルソナ一覧

### ペルソナA: 見通しを持つと動きやすい4歳児

- 基本属性: 4歳、保護者と同居、タブレット利用中心
- 目標:
  - 「次に何をするか」をすぐ理解できる
  - 終わりまでの見通しを持って安心して進める
- 困りごと:
  - 順番が曖昧だと行動が止まりやすい
  - 口頭説明だけだと内容を保持しづらい
- トリガー/ストレス要因:
  - 指示が長い、手順が途中で変わる、残り時間が見えない
- 有効な支援:
  - タスク順の固定表示
  - 視覚で減るタイマー表示
  - 1画面1メッセージの短い指示
  - 文字よりアイコンと音声を中心にした案内
- 失敗シナリオ:
  - 何を先にするか迷い、開始が遅れる
  - 終わりが見えず離席する
- 成功シナリオ:
  - タイマーを見ながら順番どおりに完了し、完了経験を積む
- 配慮事項:
  - 診断名を前提にせず、「見通し支援が必要」という行動特性で記述する

### ペルソナB: 切り替えに時間がかかる4歳児

- 基本属性: 4歳、遊びから生活行動への切り替えに課題
- 目標:
  - 遊びから生活タスクへ、無理なく移行する
  - 急な終了感を減らし、納得して行動を終える
- 困りごと:
  - 予定変更や終了指示に強い抵抗が出やすい
  - 終了直前まで時間感覚がつかみにくい
- トリガー/ストレス要因:
  - 予告なしの切り替え、終了理由が不明、達成感の欠如
- 有効な支援:
  - 残り時間の可視化
  - ごほうび時間の明示
  - 手順完了ごとの小さな達成フィードバック
  - 開始前の予告（「あと1つ遊んだらおしまい」）
- 失敗シナリオ:
  - 終了直前に反発し、次の行動に移れない
  - 保護者の再指示が増え、双方の負担が上がる
- 成功シナリオ:
  - 「終わったら次に楽しいことがある」と理解して切り替えられる
- 配慮事項:
  - 自閉スペクトラム症などの可能性を想定しつつ、個別差を前提に支援方法で記述する

### ペルソナC: 伴走する保護者

- 基本属性: 30-40代、4歳児の生活リズムを支える主担当
- 目標:
  - 毎日の声かけ回数を減らし、子どもの自走を増やす
  - 兄弟それぞれに合う設定で運用する
- 困りごと:
  - 毎回タイマー設定を作り直す手間が大きい
  - 子どもごとの差に合わせた調整が難しい
- トリガー/ストレス要因:
  - 準備に時間がかかる、失敗理由が見えない、運用負荷が偏る
- 有効な支援:
  - やることリストの再利用
  - きょうだい同時管理
  - 進行状況の一目把握
- 失敗シナリオ:
  - 設定工数が高く、アプリ運用が定着しない
- 成功シナリオ:
  - 子どもが自分で開始・完了し、保護者は最小限の介入で伴走できる
- 配慮事項:
  - 保護者の「管理負担軽減」を子どもの自立支援とセットで扱う

## 5. プロダクト判断での使い方

### 新機能追加時チェック

- 3ペルソナのうち誰のどの困りごとを改善するか明記する
- 認知負荷（読む量・選択肢数・操作回数）を増やし過ぎない
- 保護者設定コストと子どもの自走性を両立しているか確認する

### UI文言レビュー観点

- 短く具体的な文言で、次の行動が1ステップで分かるか
- 否定表現より行動提案を優先しているか
- 子ども向け表示と保護者向け設定文言の語調を分けているか

### テストケース作成観点

- 開始まで迷わないか（初回利用）
- 切り替え時に離脱しないか（終了直前）
- 中断後に再開しやすいか（保護者介入時）
