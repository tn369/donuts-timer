# ペルソナ分析

## 1. 目的と利用範囲

本ドキュメントは、「どーなつタイマー」の機能検討・UI文言設計・受け入れ基準の判断軸をそろえるためのペルソナ分析です。  
親子での利用を前提に、**4歳前後の子どもが「楽しいから自分で進めたくなる」体験を最優先**で設計します。

### 非目的

- 医療的な診断や評価の代替を行うこと
- 特定の診断名だけで利用者像を固定化すること
- 刺激を過度に強めて、子どもの行動を短期的に制御すること

## 2. 利用コンテキストの前提

- 主利用シーンは家庭内（朝の支度、帰宅後の宿題、就寝前ルーティン）
- 子ども単独操作に加え、保護者が設定や伴走を行う
- 主価値は「時間管理」そのものではなく、**短い達成体験の積み重ねで自発行動を増やすこと**
- 1セッションの中で「開始しやすい」「続けやすい」「終わりやすい」を感じられる構成を重視する
- 視覚的な進捗表示と短い文言による支援を重視する
- 子どもの個人差を前提に、刺激量（演出・情報量）は調整可能であることを重視する
- データは LocalStorage のみで管理し、外部送信しない

## 3. ペルソナ一覧

### 主ペルソナ: 達成体験で自分から動ける4歳児

- 基本属性: 4歳、保護者と同居、タブレット利用中心
- 最優先ゴール:
  - 「やってみたい」と感じて自分で開始できる
  - 小さな完了を繰り返し、最後まで進める
- 楽しさトリガー:
  - タスク完了直後に分かる称賛・演出
  - ドーナツの変化など、進捗が目で分かる動き
  - 「あと少しで次の楽しいことがある」予告
- やる気を削ぐ要因:
  - 説明が長い、次の行動が不明、達成が実感できない
- 有効な支援:
  - 1画面1アクションの短いガイド
  - アイコン中心の手順表示
  - 完了時の軽いフィードバック（視覚/音）
- 1セッション成功条件:
  - 子どもが保護者の追加指示なしで最初のタスクを開始できる
  - 完了時に次タスクへ自発的に遷移できる

### 副ペルソナ: 切り替えに予告と期待が必要な4歳児

- 基本属性: 4歳、遊びから生活行動への切り替えに時間がかかる
- ゴール:
  - 急に終わらされる感覚を減らし、納得して次へ移る
  - 「終わったら楽しいことがある」を理解して継続する
- 楽しさトリガー:
  - 終了前の段階予告（例: 3分前/1分前）
  - ごほうび時間の見通し
  - 手順完了ごとの達成反応
- やる気を削ぐ要因:
  - 予告なしの終了、終了理由が不明、最後の達成感不足
- 有効な支援:
  - 残り時間の可視化
  - ごほうび時間の明示
  - 切り替え直前の短いメッセージ
- 1セッション成功条件:
  - 終了直前の反発が減り、次行動へ移れる
  - 保護者の再指示回数が増えない

### 支援ペルソナ: 体験を壊さず準備を整える保護者

- 基本属性: 30-40代、4歳児の生活リズムを支える主担当
- 役割ゴール:
  - 子どもの楽しさを維持したまま、日常で回る設定にする
  - 兄弟それぞれに合う設定を短時間で準備する
- 困りごと:
  - 毎回設定を作り直す手間が大きい
  - 子ども差に合わせた調整ポイントが分かりづらい
- 楽しさを壊す要因:
  - 設定に時間がかかり、開始タイミングを逃す
  - 操作が増えて伴走中の声かけが増える
- 有効な支援:
  - やることリスト再利用
  - きょうだい同時管理
  - 進行状況の一目把握
- 1セッション成功条件:
  - 子どもの開始までの待ち時間を増やさず運用できる
  - 保護者介入は必要最小限で済む

## 4. 機能要件マッピング（楽しさ体験起点）

| 体験テーマ                         | 対象ペルソナ | 現状機能での対応                       | 楽しさギャップ                     | 改善アイデア                                    | 優先度 |
| :--------------------------------- | :----------- | :------------------------------------- | :--------------------------------- | :---------------------------------------------- | :----- |
| すぐ始めたくなる導入               | 主, 支援     | やることリスト、ドーナツ表示           | 開始前の期待を作る導線が弱い       | 開始時の短い前向きメッセージ、最初の一歩を明示  | 高     |
| 途中で達成感を積み上げる           | 主, 副       | ごほうびモード、進捗表示               | 完了ごとの称賛演出が限定的         | タスク完了時の軽いアニメーション/効果音の選択制 | 高     |
| 終了前の切り替えを気持ちよくする   | 副           | 残り時間表示、ごほうびモード           | 終了前の段階予告が不足             | 3分前/1分前の視覚・音予告を設定可能化           | 高     |
| 子どもに合う刺激量で続けやすくする | 主, 副, 支援 | 形状・色カスタマイズ                   | 刺激量や表示密度の調整軸が不足     | シンプル表示モード（情報量削減）                | 中     |
| 準備の手間を抑えて楽しい体験を守る | 支援         | リスト保存、きょうだいモード、複製機能 | 日々の派生編集や使い回しがまだ手間 | 曜日テンプレート、リスト派生編集フローの簡略化  | 中     |
| 時計理解に頼らず達成体験を維持する | 副, 支援     | 目標時刻モード                         | 子どもに時刻概念の負荷が高い       | 保護者向け表示と子ども向け表示の分離            | 中     |

### 優先度判定ルール

- 子どもの自発開始率を上げる効果があるか
- 切り替え時の衝突を減らせるか
- 保護者の設定・伴走の負担を増やさないか

## 5. プロダクト判断での使い方

### 新機能追加時チェック

- 3ペルソナのうち、誰のどの「楽しさギャップ」を埋めるかを明記する
- 子どもが「やりたい」と感じる瞬間を、画面上のどこで作るかを明記する
- 認知負荷（読む量・選択肢数・操作回数）を増やし過ぎない
- 保護者設定コストを増やす場合、削減策または上限を合わせて定義する

### UI文言レビュー観点

- 短く具体的で、次の行動が1ステップで分かるか
- 否定表現より「できた」「つぎが楽しみ」につながる提案表現を優先しているか
- 子ども向け文言は達成感を即時に返し、保護者向け文言は設定意図が短く伝わるか

### テストケース作成観点

- 開始まで迷わないか（初回利用）
- 完了時に次タスクへ進みたくなるか（達成直後）
- 切り替え時に離脱しないか（終了直前）
- 中断後に再開しやすいか（保護者介入時）
